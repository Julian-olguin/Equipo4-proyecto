#  Proyecto de Práctica: Gestión de Versiones con GitHub Desktop

## Objetivo del Proyecto
Este repositorio tiene como finalidad poner en practica el flujo de trabajo colaborativo en Git. El equipo simulará un entorno de desarrollo real donde se crearan archivos de datos y scripts de analisis, se generarán conflictos intencionales y se administrará el historial de versiones.

##  Reglas de Colaboración (Normativa)
Para mantener el orden, todos los integrantes deben seguir estas reglas:
* **Commits:** Usar mensajes claros y descriptivos en GitHub Desktop.
* **Código:** Todo cambio en el código debe incluir un comentario con:
    * Iniciales del autor.
    * Fecha de modificación.
    * Explicación breve del cambio.
* **Conflictos:** Los conflictos de versiones serán resueltos exclusivamente por el Administrador.
